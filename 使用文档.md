## 一、先看这一节：新手最快上手方法

如果你想尽快用起来，只要按下面顺序做：

1. 打开软件，先输入卡密完成登录
2. 进入“账户管理”，添加你的邮箱账号并先测试通过
3. 进入“模型设置”，添加至少 1 个可用模型并测试通过
4. 进入“设置”，确认抓取天数、抓取文件夹、下载目录
5. 回到“仪表盘”，点击“开始分析”
6. 到“邮件结果”里查看高价值邮件和退订邮件
7. 如果要分析电脑里的邮件文件，进入“本地邮件”页面操作

如果你只想分析本地邮件文件，不连接邮箱账号也可以，但仍然建议先配置好 AI 模型。

## 二、软件是做什么的

AI 邮件智能分析是一款帮助你批量整理邮箱邮件的工具，主要作用是：

- 自动读取邮箱中的邮件
- 用 AI 判断哪些邮件更值得重点跟进
- 自动识别退订邮件
- 把分析结果汇总到结果页和统计页
- 支持查看来往邮件
- 支持 AI 生成回复内容
- 支持发送附件
- 支持分析本地 `.eml`、`.mbox`、`.msg` 邮件文件
- 支持导出结果为 Excel、CSV、TXT
- 支持根据知识库规则提升分析准确率

适合这些使用场景：

- 外贸开发信筛选
- 询盘筛选
- 批量清理垃圾邮件、广告邮件、退信邮件
- 多邮箱统一管理
- 已下载邮件文件的离线整理

## 三、使用前需要准备什么

### 1）卡密

软件首次使用时，需要卡密验证。

说明：

- 验证成功后才能进入软件
- 卡密会绑定当前电脑设备 ID
- 验证成功后，后续一般不需要每次重新输入
- 软件每天会自动校验一次授权状态
- 如果卡密到期，分析功能会被限制

### 2）邮箱账号信息

如果你要分析在线邮箱，请提前准备：

- 邮箱地址
- 邮箱密码或授权码
- IMAP 服务器地址
- IMAP 端口
- SMTP 服务器地址
- SMTP 端口

很多邮箱不能直接用登录密码，而要用“授权码”。如果测试失败，先去邮箱官网确认是否已经开启 IMAP / SMTP。

### 3）AI 模型接口信息

如果你要让 AI 帮你判断邮件价值、生成回复内容，就必须先配置模型。

通常需要准备：

- 模型名称
- API Base
- API Key

填写时请注意：

- API Base 一般填写到 `/v1`
- 例如：`https://api.openai.com/v1`
- 如果文档里写的是 `/v1/chat/completions`，这里通常只填到 `/v1` 即可

### 4）如果你要分析本地邮件文件

请提前准备一个文件夹，并把邮件文件放进去。软件支持：

- `.eml`
- `.mbox`
- `.msg`

建议把要分析的文件单独放到一个目录中，这样更容易管理。

- 如果之前已经验证成功，软件通常会自动跳过登录界面

- 本地邮件：分析本地电脑中的邮件文件
- 下载管理：管理下载的邮件内容或附件

- 设置：配置抓取范围、自动回复、通知、主题、定时运行等

## 八、账户管理怎么用

如果你要分析在线邮箱，先进入“账户管理”。

### 1）你可以在这里做什么

账户管理支持：

- 手动添加账号
- 批量导入账号
- 测试选中账号
- 批量启用 / 禁用
- 批量删除
- 导出账号信息
- 修改单个账号的 IMAP / SMTP 设置

### 2）一个账号至少要准备哪些内容

通常需要：

- 邮箱
- IMAP 服务器
- IMAP 端口
- SMTP 服务器
- SMTP 端口
- 密码或授权码
- 状态（启用或禁用）

### 4）最推荐的操作方法

建议你这样做：

1. 先添加 1 个账号
2. 点击“测试选中”或页面中的测试入口
3. 测试通过后，再继续添加其他账号
4. 全部测试通过后，再开始正式分析

这样能最快发现问题。

### 5）如何批量导入账号

支持三种方式：

- 文本导入
- JSON 导入
- 拖拽文件导入

文本导入格式示例：

```text
邮箱 密码 IMAP服务器 端口 SMTP服务器 SMTP端口
```

如果你只写“邮箱 + 密码”，软件会尽量使用页面上方的默认服务器设置帮你补全。

### 6）页面上的默认服务器有什么用

账户导入页面上方可以填写默认 IMAP / SMTP 和端口。

它的作用是：

- 你批量导入时少填一些字段
- 相同服务商的邮箱可以更快导入
- 可以通过“同域推断 SMTP”快速补全

### 7）如何看测试结果

账户列表中的测试结果通常表示：

- 通过：账号配置基本正常
- 失败：账号信息或服务器配置有问题
- 未测试：还没有验证

如果测试失败，最常见原因是：

- 邮箱密码错误
- 填写了登录密码，但邮箱实际要求授权码
- IMAP 没开启
- SMTP 没开启
- 服务器地址写错
- 端口写错
- 邮箱服务商启用了额外安全限制

## 九、模型设置怎么用

模型设置是整个软件能否正常进行 AI 分析的关键页面。

### 1）至少要做什么

你至少要保证：

- 配置 1 个模型
- 这个模型是启用状态
- 测试结果为通过

### 2）每个模型需要填写什么

一般需要：

- 模型名称
- API Base
- API Key
- 启用状态

### 3）推荐做法

推荐至少准备 2 个模型：

- 主模型：平时优先使用
- 备用模型：主模型失败时可以自动切换

### 4）标准配置步骤

1. 点击“新增模型”
2. 填写模型名称
3. 填写 API Base
4. 填写 API Key
5. 勾选启用
6. 点击“测试选中”
7. 确认通过后点击“保存修改”

### 5）怎么看模型是否正常

模型列表中一般会显示：

- 是否启用
- 测试结果
- 最近说明
- 累计 Token

含义说明：

- 通过：模型接口连接正常
- 失败：接口地址、密钥或网络有问题
- 累计 Token：该模型历史消耗的 token 数量统计

### 6）新手最容易出错的地方

最常见错误是 API Base 填太长。

例如文档里如果写的是：

```text
https://xxx.com/v1/chat/completions
```

你在软件里通常应该填写：

```text
https://xxx.com/v1
```

## 十、设置页面怎么配置

对于新手来说，“设置”页面最重要的是这几个选项。

### 1）抓取最近天数

表示分析最近几天收到的邮件。

新手推荐：

- 先填 2 天或 3 天
- 不要一开始就设置太大

这样速度更快，也方便先验证效果。

### 2）抓取文件夹

默认推荐保留：

```text
INBOX,SPAM,Junk,Spam,垃圾邮件
```

意思是：

- 收件箱会分析
- 常见垃圾邮件文件夹也会分析

如果你的邮箱文件夹名称不同，可以按实际情况修改。

### 3）默认下载目录

这是邮件附件或下载内容的保存位置。

建议选择一个固定文件夹，后期更方便查找。

### 4）界面主题

可以选择：

- light
- dark

按个人习惯选择即可。

### 5）监听轮询间隔

表示持续分析模式下，每次分析完后多久再次检查是否有新邮件。

如果你只是手动分析一次，先保持默认值即可。

### 6）分析任务并发处理账号

如果你添加了很多邮箱账号，可以开启并发处理。

新手建议：

- 先关闭或使用较小并发数
- 等确认稳定后，再逐步增加

### 7）自动回复

如果你想启用自动回复，可以在设置页中配置：

- 开启 AI 自动回复，首次回复
- 自动回复成功后发送通知
- 通知发件账号
- 通知收件人
- 自动回复要求

“自动回复要求”里可以直接写自然语言，例如：

- 回复语气正式一些
- 尽量简短
- 先感谢再介绍产品
- 回复用英文
- 不要主动报价

### 8）定时运行

你可以设置：

- 每天固定时间自动开始分析
- 或者每隔 N 分钟自动运行一次

注意：

- 间隔为 0：表示按“每日启动时刻”每天到点运行一次
- 间隔大于等于 1：表示每隔 N 分钟运行一次

如果你是第一次使用，建议先关闭定时，等手动分析稳定后再开启。

### 9）通知邮件怎么理解

这里的“通知”主要是指：

- 自动回复成功后，系统可以再发一封通知邮件给你指定的邮箱

如果你暂时不需要这个功能：

- 不开启“自动回复成功后发送通知”即可

## 十一、知识库怎么用

知识库是帮助 AI 更懂你的业务、更少误判的重要页面。

### 1）你可以放什么进去

常见内容包括：

- 产品介绍
- 竞争对手信息
- 回复模板
- 自动回复签名
- 黑名单关键词
- 跳过规则
- 通用说明
- 正向学习样本
- 负向学习样本

### 2）最实用的四种内容

对于大多数用户，最有用的是这四类：

#### 产品介绍

把你的产品、服务、客户类型写进去，让 AI 更容易判断哪些询盘是真正有价值的。

#### 回复模板

把你常用的商务回复、英文模板、报价前置话术写进去，后面生成回复草稿会更贴近你的风格。

#### 自动回复签名

把公司名、联系人、电话、网站、落款等写在这里。后续自动回复或 AI 回复生成时，可以更容易带上固定签名。

#### 跳过规则

把明显没必要分析的邮件规则写进去，可以减少无意义的 AI 调用。

### 3）跳过规则怎么写

支持的常见写法有：

- 发件人
- 发件人包含
- 主题
- 主题包含
- 内容包含

示例：

```text
发件人: no-reply@example.com
发件人包含: mailer-daemon
主题: Mail delivery failed
主题包含: Delivery Status Notification
内容包含: unsubscribe
```

你也可以在一行里写多个值，支持这些分隔方式：

- 逗号
- 分号
- 中文逗号
- “或”

例如：

```text
发件人包含: mailer-daemon 或 postmaster
主题包含: Delivery Status Notification, Returned mail
内容包含: unsubscribe；click here to unsubscribe
```

命中跳过规则后：

- 这封邮件会直接跳过
- 不再调用 AI 分析

### 4）黑名单关键词和跳过规则有什么区别

这两个很容易混淆。

区别很简单：

- 黑名单关键词：是让 AI 更倾向判为无用邮件
- 跳过规则：是程序直接跳过，不再分析

如果你想“绝对不分析”，就用“跳过规则”。

### 5）知识库怎么添加最简单

你可以按下面方法操作：

1. 点击“新建”
2. 填写标题
3. 选择类型
4. 在“内容”里粘贴正文
5. 状态保持为 `enabled`
6. 点击“保存片段”

如果你已经有整理好的文本文件，也可以用“上传 .txt / .md”直接导入。

### 6）启用和禁用是什么意思

只有启用状态的知识片段，分析时才会真正参与判断。

所以：

- 想暂时停用某条规则，不用删，直接改成禁用就行
- 批量规则整理时，可以先导入，再慢慢启用

### 7）学习样本是什么

当你在结果页里点击：

- `✔` 表示判断正确
- `✖` 表示判断错误

系统会把这些反馈逐步写入学习样本，帮助后续判断更贴近你的实际标准。

如果你在知识库类型里看到：

- `learning_positive`
- `learning_negative`

它们就是学习样本类型。

## 十二、仪表盘怎么用

仪表盘是你每天最常看的页面。

这里通常可以看到：

- 当前版本号
- 通知按钮
- 授权到期时间
- 当前授权状态
- 开始分析 / 暂停分析按钮
- 查看日志按钮
- 自动回复开关
- 进度条
- 运行日志
- 总分析数、有效邮件数、退订邮件数、Token 等统计卡片

### 1）开始分析怎么用

操作很简单：

1. 确保账户和模型都已经配置好
2. 回到仪表盘
3. 点击“开始分析”
4. 观察状态、进度条和运行日志

如果运行中需要停下来，再点击暂停即可。

### 2）查看日志有什么用

如果你遇到这些情况，先看日志：

- 一直没有结果
- 账号连不上
- 模型调用失败
- 某个邮箱处理异常

日志是最直接的排查入口。

### 3）通知按钮怎么用

通知按钮可能显示两种状态：

- `通知`
- `有新版本`

如果显示“有新版本”，说明软件检测到远程版本比本地版本新。

你可以点击它查看：

- 最新版本号
- 更新通知内容
- 下载地址

### 4）卡密过期时间在哪里看

仪表盘右上区域会显示：

- 授权到期时间
- 当前授权状态
- 卡密输入框
- “验证”按钮

如果需要重新校验卡密，可以直接在这里操作。

## 十三、邮件结果怎么看

“邮件结果”页面是日常处理邮件最重要的页面。

它分成两个主要标签：

- 有效邮件
- 退订请求

## 十四、有效邮件页面怎么用

### 1）顶部筛选区有什么用

有效邮件页面上方可以做这些事：

- 搜索发件人、主题、摘要、备注
- 按行业筛选
- 按价值等级筛选
- 按国家/地区筛选
- 选择导出模式
- 导出选中
- 导出全部结果
- 放大结果
- 删除选中

其中“导出模式”支持：

- 导出为扁平表格
- 按行业分 Sheet（Excel）
- 按国家分 Sheet（Excel）

### 2）结果列表里主要看什么

常见列包括：

- 发件人
- 主题
- AI 摘要
- 判断依据
- 行业
- 价值等级
- 国家/地区
- 最新回信时间
- 备注
- 回复状态
- 通知状态
- 人工判定
- 操作

如果发件人前面有红点，一般表示：

- 未读
- 或者有新回复

如果前面有图钉，表示这条邮件已被置顶。

### 3）备注怎么写

备注列支持手动编辑。

建议你把这些信息写进去：

- 客户意向
- 当前跟进状态
- 价格沟通情况
- 是否已报价
- 下次联系时间

### 4）如何人工纠正 AI 判断

每行里通常会有两个反馈按钮：

- `✔`：确认这条判断是对的
- `✖`：认为这条判断是错的

这些反馈会进入学习样本，后续可帮助 AI 更贴近你的标准。

### 5）如何导出结果

如果你只想导出部分记录：

- 先选中要导出的行
- 点击“导出选中”

如果你想一次性导出当前列表：

- 点击“导出全部结果”

支持导出为：

- Excel
- CSV
- TXT

### 6）放大结果怎么用

如果你觉得表格区域太小，可以点击“放大结果”。

放大窗口中支持：

- 搜索当前结果
- 复制选中行
- 导出当前视图
- 查看往来
- 右键置顶 / 下载 / 查看会话

适合大屏筛选和集中处理。

### 7）自动拉取是什么

页面上方有“自动拉取：开启/关闭”和拉取间隔。

它的作用是：

- 在结果页持续检查新的来往回复
- 自动更新未读红点和最新回信时间

如果你暂时不需要实时更新，可以把它关闭。

## 十五、查看往来和快速回信怎么用

这是很多用户最常用的功能。

### 1）怎么打开往来记录

打开方法有几种：

- 在结果列表中点击“查看往来”
- 在放大结果窗口中点击“查看往来”
- 在结果表格上点右键，选择“查看往来信件”

### 2）往来窗口里可以做什么

往来窗口通常支持：

- 查看同一客户的邮件记录
- 切换“仅当前主题”
- 切换“全部历史”
- 点击“拉取邮件”刷新记录
- AI 生成回信草稿
- 插入图片或文件附件
- 发送回信

### 3）AI 回复草稿怎么生成

操作步骤：

1. 选中一封有效邮件
2. 打开“查看往来”或直接在主结果页右侧详情区操作
3. 点击“生成 AI 回复草稿”或“AI生成回复”
4. 等待草稿生成
5. 人工修改后再发送

建议不要完全不看就直接发，最好快速检查一遍措辞和附件。

### 4）怎么发送附件

无论在主结果页还是往来窗口里，都可以：

1. 点击“插入图片/文件”
2. 选择一个或多个附件
3. 确认附件名称已显示
4. 点击“发送邮件”或“发送回信”

### 5）发送邮件需要满足什么条件

要成功发送邮件，至少要满足：

- 账户管理里有可用的 SMTP 服务器和端口
- 账号密码或授权码正确
- 网络正常

如果发送失败，优先检查：

- SMTP 配置
- 授权码
- 端口是否正确

## 十六、退订请求页面怎么用

这个页面专门整理被识别为退订意图的邮件。

### 1）你可以做什么

支持：

- 搜索发件人、主题、正文
- 复制选中的发件人
- 导出选中记录
- 删除选中记录
- 人工确认正确或错误
- 查看完整正文

### 2）复制发件人有什么用

当你想批量整理退订名单、黑名单名单或者广告发件地址时，这个功能很方便。

### 3）如何纠正误判

每条退订记录也有：

- `✔`：确认它确实是退订邮件
- `✖`：说明它不应该归到退订里

你的反馈会进入学习样本，后续识别会更贴近你的需求。

### 4）退订页支持哪些导出格式

通常支持：

- CSV
- Excel

## 十七、本地邮件怎么用

如果你电脑里已经有很多邮件文件，可以用“本地邮件”页面离线分析。

### 1）支持哪些格式

支持：

- `.eml`
- `.mbox`
- `.msg`

### 2）标准操作步骤

1. 进入“本地邮件”页面
2. 点击“选择邮件目录…”
3. 如果你的邮件文件分布在子文件夹里，勾选“递归扫描子目录”
4. 点击“开始本地分析”
5. 在下方日志区查看进度
6. 完成后去“邮件结果”和“统计分析”查看结果

### 3）日志里会显示什么

日志里通常会显示：

- 开始分析了哪个文件
- 是否跳过重复文件
- 是否命中跳过规则
- 是否分析失败
- 最终汇总数量

### 4）本地分析和在线分析的关系

本地分析的结果会写入现有：

- 邮件结果页
- 统计页

也就是说，你不需要另外看一个新页面，结果会和日常结果放在一起。

### 5）本地分析适合什么场景

适合这些情况：

- 手里有客户发来的 `.eml` 邮件样本
- 从别的软件导出过邮件文件
- 想先离线筛选一批历史邮件
- 不想直接连接邮箱账号

### 6）需要注意什么

- 本地邮件分析也需要可用 AI 模型
- 如果只是离线整理，不一定需要邮箱账号
- 但如果你后面还要直接发送回信，仍然需要在“账户管理”里配置 SMTP

## 十八、统计分析怎么用

统计分析页面适合看整体趋势，而不是看单封邮件。

### 1）这个页面能看到什么

按日期查看：

- 有效邮件
- 退订
- 系统退信
- 被拦截

### 2）原因统计怎么用

当你选中某一天或多天后，下方会显示：

- 退信原因统计
- 被拦截原因统计

这对排查邮箱质量、投递问题很有帮助。

### 3）一键分析有什么用

点击“一键分析”后，AI 会结合统计数据给出：

- 趋势判断
- 退信高的主要原因
- 被拦截高的主要原因
- 优先级建议
- 后续监控建议

如果你平时不知道怎么从数字里看问题，这个功能很有用。

### 4）统计保留天数是什么

你可以设置保留多少天的统计数据。

如果你点击“保存并清理”，系统会按你设置的保留天数清理旧统计。

新手建议先保持默认值，不要频繁改动。

## 十九、下载管理怎么用

下载管理用于查看邮件下载任务和历史记录。

### 1）怎么进入

左侧点击“下载管理”即可打开下载窗口。

### 2）下载管理里能做什么

支持查看：

- 文件名
- 下载状态
- 进度条
- 保存路径
- 打开文件
- 打开文件夹

### 3）怎么下载某封邮件

在有效邮件列表或放大结果中，通常可以通过右键菜单选择：

- `下载邮件 (.eml)…`

下载后的文件会进入你在“设置”里指定的默认下载目录。

### 4）什么时候会下载失败

如果某条记录缺少原始账号或邮件 UID，可能无法下载 `.eml` 原件。

一般来说：

- 在线邮箱分析得到的邮件，更适合下载原始 `.eml`
- 纯本地导入分析的邮件，不一定具备在线下载条件

## 二十、WhatsApp 页面怎么用

这个页面属于可选功能，不是日常邮件分析必填项。

如果你暂时不用 WhatsApp，可以先忽略本页，不影响邮件分析。

### 1）它是做什么的

主要用于：

- 配置 Webhook
- 接收 WhatsApp 消息
- 使用 AI 和知识库进行被动回复

### 2）最简单的理解方式

你可以把它理解成：

- 别人先给你发消息
- 系统收到后再按规则进行回复

### 3）页面里主要项怎么填

常见项包括：

- Host
- Port
- Webhook Path
- Verify Token
- 发送接口 URL
- 发送接口 Token

### 4）基本操作步骤

1. 填写 Host / Port / Webhook Path / Verify Token
2. 如有需要，再填写发送接口 URL / Token
3. 勾选启用
4. 点击“保存 WhatsApp 设置”
5. 使用“复制地址”复制 Webhook 地址
6. 到你的对接平台粘贴这个地址

如果你只是普通邮件用户，不接 WhatsApp，完全可以跳过这页。

## 二十一、最常见问题

### 1）卡密验证失败怎么办

先检查：

- 卡密是否输错
- 网络是否正常
- 卡密是否已过期
- 当前电脑是否已绑定其他卡密信息

### 2）邮箱测试失败怎么办

优先检查：

- 是不是应该填授权码而不是登录密码
- IMAP / SMTP 是否开启
- 服务器地址和端口是否正确
- 邮箱服务商是否有限制

### 3）模型测试失败怎么办

优先检查：

- API Base 是否只填到 `/v1`
- API Key 是否正确
- 网络是否能访问模型接口
- 该接口是否兼容 OpenAI 风格

### 4）点击开始分析后没结果怎么办

按这个顺序检查：

1. 打开仪表盘“查看日志”
2. 确认账户至少有 1 个启用并测试通过
3. 确认模型至少有 1 个启用并测试通过
4. 检查“抓取最近天数”是不是太小
5. 检查“抓取文件夹”是否包含真实邮件所在文件夹

### 5）能分析但不能发送邮件怎么办

这通常是 SMTP 配置问题。

请检查：

- 账户里是否填写了 SMTP 服务器
- SMTP 端口是否正确
- 密码 / 授权码是否正确
- 是否需要 SSL 或 TLS

### 6）本地邮件没有分析出结果怎么办

优先检查：

- 文件夹里是否真的有 `.eml`、`.mbox`、`.msg`
- 是否勾选了“递归扫描子目录”
- 模型是否可用
- 日志里是否提示内容为空、重复文件、命中跳过规则或模型失败

### 7）为什么有些邮件被直接跳过

常见原因有：

- 命中了知识库里的“跳过规则”
- 是重复文件
- 邮件内容为空
- 不在最近抓取天数范围内

### 8）导出的 CSV 打开后中文乱码怎么办

软件导出的 CSV / TXT 已尽量使用适合中文的编码。

如果你打开后仍觉得显示不正常，建议：

- 优先导出 Excel `.xlsx`
- 或者在 Excel 中用“导入文本/CSV”的方式选择 UTF-8 打开
- 不要用不支持 UTF-8 的老旧编辑器直接打开

## 二十二、新手每天最稳妥的使用流程

如果你已经完成配置，日常可以按这个流程使用：

1. 打开软件，看仪表盘授权状态是否正常
2. 点击“开始分析”
3. 看日志是否有明显报错
4. 进入“邮件结果”优先处理高价值邮件
5. 对需要跟进的邮件查看往来并生成回复草稿
6. 有必要时导出结果给同事或老板
7. 最后看一下“统计分析”，了解当天整体情况

## 二十三、给新手的最后建议

为了更稳妥地使用，建议你记住下面几点：

- 第一次使用时，先只加 1 个邮箱、1 个模型，测试通了再扩大
- 第一次分析时，抓取最近天数先不要太大
- 自动回复不要一上来就全开，先手动检查效果
- 重要回复先看一遍再发送，不要完全依赖自动生成
- 知识库不要一次堆太多内容，先从产品介绍、跳过规则、签名开始
- 导出给别人时，优先使用 Excel 格式，更不容易出现中文显示问题

如果你完全不懂原理，也没有关系。

只要你按照这份文档的顺序操作，先把：

- 账户配置好
- 模型配置好
- 设置检查好

这三步完成，后面的使用就会顺畅很多。
